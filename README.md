# Тропа — авторские экскурсии

## Что внутри

- **index.html** — весь клиент (HTML+CSS+JS), данные хранятся в `localStorage` браузера. Открывается просто двойным кликом или через любой статический сервер.
- **server.js / package.json / .env.example** — маленький сервер на Node.js + Nodemailer, который умеет реально слать письма (подтверждение брони, контакты гида после одобрения).

В `index.html` письма пока не отправляются взаправду — они падают в раздел «Почта (демо)» внутри приложения, чтобы можно было проверить весь сценарий без настройки почты. Ниже — как подключить настоящую отправку.

## Как завести почту для рассылки и подключить её

Отправлять письма от имени пользовательского Gmail/Яндекс-ящика напрямую из браузера **нельзя** — почтовые провайдеры не разрешают SMTP из JS на клиенте, да и пароль в открытом виде во фронтенде — плохая идея. Поэтому письма шлёт `server.js`, а не `index.html`.

### Вариант 1 — Gmail (проще всего, бесплатно, до ~500 писем/сутки)

1. Заведите отдельный Google-аккаунт для проекта, например `tropa.tours.app@gmail.com` (не используйте личную почту).
2. Включите двухфакторную аутентификацию: **Google-аккаунт → Безопасность → Двухэтапная аутентификация**.
3. Там же откройте **«Пароли приложений»** (App passwords) — этот пункт появляется только после включения 2FA. Создайте пароль приложения для «Почта» / «Другое устройство» — Google даст 16-значный код.
4. Скопируйте `.env.example` в `.env` и впишите:
   ```
   SMTP_HOST=smtp.gmail.com
   SMTP_PORT=465
   SMTP_USER=tropa.tours.app@gmail.com
   SMTP_PASS=16-значный пароль приложения (не обычный пароль от аккаунта!)
   ```
5. Готово — обычный пароль от аккаунта тут не подойдёт, нужен именно пароль приложения.

### Вариант 2 — Яндекс.Почта (тоже бесплатно, удобно для РФ)

1. Заведите ящик на yandex.ru для проекта.
2. **Настройки → Почтовые программы → Пароль приложения** — включите доступ по протоколу IMAP/SMTP и создайте пароль приложения так же, как в Gmail.
3. В `.env`:
   ```
   SMTP_HOST=smtp.yandex.ru
   SMTP_PORT=465
   SMTP_USER=tropa.tours@yandex.ru
   SMTP_PASS=пароль приложения
   ```

### Вариант 3 — сервис транзакционных писем (Resend / SendGrid / Mailgun)

Если писем будет много (десятки-сотни в день) или нужна аналитика доставки — лучше не гонять их через личный Gmail/Яндекс, а завести аккаунт в сервисе для транзакционных писем:

- **Resend** (resend.com) — самый простой старт, бесплатный тариф, есть готовый SMTP и SDK для Node.js.
- **SendGrid**, **Mailgun** — аналогично, чуть больше настройки, зато выше бесплатные лимиты и есть свой домен для писем (например, `mail@tropa-tours.ru`), что снижает шанс попасть в спам.

Для любого из них процесс одинаковый: регистрируетесь → получаете SMTP-логин/пароль или API-ключ → вписываете в `.env` (для Resend/SendGrid можно использовать их SMTP-реле почти без изменений в `server.js`, просто заменить `SMTP_HOST`/`SMTP_USER`/`SMTP_PASS` на выданные сервисом).

## Запуск сервера писем

```bash
npm install
cp .env.example .env   # и вписать туда свои данные
npm start
```

При старте в консоли появится либо `SMTP подключен, можно слать письма`, либо ошибка — сразу будет понятно, если логин/пароль неверные.

## Как подключить фронтенд к серверу

Сейчас в `index.html` есть функция:

```js
function sendMail(to, subject, body){
  const mail = loadMail();
  mail.unshift({id:uid(), to, subject, body, ts:new Date().toISOString()});
  saveMail(mail);
}
```

Чтобы письма реально уходили, замените её тело на запрос к серверу (оставив запись в демо-почту как резерв на случай ошибки сети):

```js
async function sendMail(to, subject, body){
  try{
    await fetch('http://localhost:3001/api/send-mail', {
      method: 'POST',
      headers: {'Content-Type':'application/json'},
      body: JSON.stringify({to, subject, body})
    });
  }catch(e){
    console.error('Не удалось отправить письмо, смотрите демо-почту', e);
  }
  const mail = loadMail();
  mail.unshift({id:uid(), to, subject, body, ts:new Date().toISOString()});
  saveMail(mail);
}
```

При деплое в интернет замените `http://localhost:3001` на адрес вашего сервера, и добавьте туда же настоящую базу данных вместо `localStorage` (сейчас это временное локальное хранилище для одного браузера — как и просили в задании как минимальный вариант, но за полноценную БД, по условию, дают доп. баллы).

## Как проверить весь сценарий без настройки почты

1. Откройте `index.html`.
2. Выберите экскурсию → «Забронировать» → пройдите шаги → оплатите (кнопка-заглушка).
3. Откройте вкладку «Почта (демо)» — там письмо о подтверждении брони.
4. Откройте «Мои экскурсии», введите email создателя экскурсии (для тестовых экскурсий — `irina.guide@example.com`, `maxroofs@example.com` или `anna.audio@example.com`), одобрите заявку.
5. Снова откройте «Почта (демо)» — появится письмо с контактами гида.

## Соответствие условиям задания

- Веб-клиент + локальное хранилище (`localStorage`) реализованы; для доп. баллов — подключите БД (Postgres/SQLite) и Node-сервер вместо `localStorage`, а также при желании мобильный клиент.
- Главная страница: список экскурсий по текущему городу пользователя + фильтр (страна, город, даты, кол-во человек, дети) с кнопкой «Поиск».
- Карточка экскурсии: название, фото, краткое описание, формат, длительность, цена.
- Бронирование: выбор времени (если применимо) → оплата (онлайн-заглушка / на месте) → подтверждение на почту → контакты гида на почту после одобрения.
- Создание экскурсии доступно любому пользователю через кнопку в правом нижнем углу, форма содержит все поля из задания.
